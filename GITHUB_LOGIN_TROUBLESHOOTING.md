# GitHubログイン トラブルシューティングガイド

## 🔍 考えられる原因と対処法

### 1. ユーザー名とメールアドレスの混同

GitHubでは、**ユーザー名（username）**でログインします。メールアドレスでもログインできる場合がありますが、組織のSSO認証が有効な場合はユーザー名が必要な場合があります。

**確認事項**:
- `developer.kops1124` → **ユーザー名（username）**の可能性が高い
- `developer@ko-partners.biz` → **メールアドレス**

**対処法**:
1. まず `developer.kops1124` でログインを試してください
2. パスワードは正しいですか？（他の担当者に確認）
3. メールアドレスでログインする場合は、GitHubの設定でメールアドレスでのログインが有効になっている必要があります

### 2. パスワードの問題

**確認事項**:
- パスワードの入力ミスがないか
- Caps Lockがオンになっていないか
- パスワードが変更されていないか（他の担当者に確認）

**対処法**:
- 「パスワードを忘れた場合」からリセットを試す
- 他の担当者に現在のパスワードを確認

### 3. 2要素認証（2FA）の問題

組織のアカウントでは2要素認証が有効になっている可能性があります。

**確認事項**:
- 2FAが有効になっているか
- 認証アプリ（Google Authenticator、Authyなど）が設定されているか
- SMS認証が設定されているか

**対処法**:
- 2FAの認証コードを正しく入力
- 認証アプリが正しく同期されているか確認
- バックアップコードがあるか確認

### 4. 組織のSSO認証が必要

JPDigitalCoLtd組織のリポジトリにアクセスするには、組織のSSO認証が必要な場合があります。

**確認事項**:
- 通常のGitHubログイン後に、組織の認証ページにリダイレクトされるか
- 「Single sign-on organizations」のポップアップが表示されるか

**対処法**:
1. まず通常のGitHubログインを完了
2. その後、組織のSSO認証ページで認証を完了
3. 組織の認証システム（例: Azure AD、Oktaなど）で認証が必要な場合がある

### 5. ブラウザの問題

**対処法**:
1. **シークレット/プライベートモードで試す**
   - Chrome: `Cmd + Shift + N`
   - Safari: `Cmd + Shift + N`
   - Firefox: `Cmd + Shift + P`

2. **キャッシュとCookieをクリア**
   - Chrome: 設定 → プライバシーとセキュリティ → 閲覧履歴データの削除
   - Safari: 環境設定 → プライバシー → すべてのWebサイトデータを削除

3. **別のブラウザで試す**

### 6. アカウントの状態確認

**確認事項**:
- アカウントが凍結されていないか
- アカウントが削除されていないか
- 組織から削除されていないか

**対処法**:
- GitHubサポートに問い合わせ: https://support.github.com/

## 🔐 推奨されるログイン手順

### ステップ0: ユーザー名の確認

まず、ユーザー名が正しいか確認してください：
- https://github.com/developer.kops1124 にアクセス
- プロフィールページが表示されれば、ユーザー名は正しいです
- 404エラーが出る場合は、ユーザー名が間違っている可能性があります

### ステップ1: ユーザー名でログイン

1. https://github.com/login にアクセス
2. **ユーザー名**: `developer.kops1124` を入力（メールアドレスではなく）
3. **パスワード**: （他の担当者に確認）
4. 「Sign in」をクリック

**重要**: メールアドレス（`developer@ko-partners.biz`）ではなく、**ユーザー名（`developer.kops1124`）**でログインしてください。

### ステップ2: 2FA認証（必要な場合）

1. 2FAの認証コードを入力
2. 認証アプリまたはSMSからコードを取得

### ステップ3: 組織のSSO認証（必要な場合）

1. ログイン後、組織の認証ページにリダイレクトされる場合がある
2. 組織の認証システムで認証を完了
3. 「Authorize」または「Continue」をクリック

## 🆘 エラーメッセージ別の対処法

### "Incorrect username or password"
- ユーザー名とパスワードを再確認
- 他の担当者に正しい認証情報を確認

### "Account has been disabled"
- GitHubサポートに問い合わせが必要

### "SAML SSO required"
- 組織のSSO認証が必要
- 組織の認証システムで認証を完了

### "Two-factor authentication required"
- 2FAの認証コードを入力
- 認証アプリが正しく設定されているか確認

## 💡 代替案: 個人アカウントでアクセス権限を付与してもらう

もし組織アカウントでログインできない場合、以下の方法も検討できます：

1. **個人のGitHubアカウントを使用**
   - 個人アカウントでログイン
   - 組織の管理者に、個人アカウントへのアクセス権限を付与してもらう

2. **GitHub Personal Access Token (PAT) を使用**
   - 他の担当者がPATを生成して共有
   - HTTPSクローン時にPATをパスワードとして使用

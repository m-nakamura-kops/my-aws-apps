# 管理者画面 403 解消後の確認手順

管理者画面（イベント管理・生徒名簿・スタッフ管理・レポート出力）で「User email is required」による 403 を解消する修正を入れたあと、次の手順で動作を確認してください。

---

## 1. バックエンドの local-server を再起動する

1. バックエンドの local-server が動いているターミナルで **Ctrl+C** を押し、一度停止する。
2. 再度起動する。

```bash
cd apps/qr-attendance/backend
npm run dev
```

（または `cd apps/qr-attendance/backend && npm run dev`。矢印「→」は使わず、半角の `&&` でつなげてください）

3. ターミナルに `Listening on port 3001` などと表示され、エラーなく待機していることを確認する。

---

## 2. フロントを再読み込みする

1. ブラウザで **http://localhost:3000** を開いているタブに移動する。
2. **スーパーリロード** する（キャッシュを無視して再読み込み）。
   - **Windows / Linux**: `Ctrl + Shift + R` または `Ctrl + F5`
   - **macOS**: `Cmd + Shift + R`
3. 必要なら、一度 **ログアウト** してから次に進む。

---

## 3. 管理者でログインし直す

1. ログイン画面で **管理者アカウント** でログインする。
   - 例: `admin@example.com` / `admin123`（スクリプトで作成した場合）
2. ダッシュボード（ホーム）が表示されることを確認する。

---

## 4. 各管理者画面で 403 が解消しているか確認する

次の各画面を開き、**403 エラーや「User email is required」のメッセージが出ず、一覧などが表示されること**を確認してください。

| 画面           | URL                          | 確認内容                                   |
|----------------|------------------------------|--------------------------------------------|
| イベント管理   | http://localhost:3000/admin/events   | 一覧が表示される（イベントがなくても「イベントがありません」など正常表示） |
| 生徒名簿管理   | http://localhost:3000/admin/students | 一覧が表示される                           |
| スタッフ管理   | http://localhost:3000/admin/staffs   | 一覧が表示される                           |
| レポート出力   | http://localhost:3000/admin/reports  | イベント一覧または「イベントがありません」が表示される |

- 画面上部の **赤いエラーバナー** が出ていないこと。
- ブラウザの **開発者ツール（F12）→ Console** に **403 (Forbidden)** が出ていないこと。

---

## うまくいかない場合

- **まだ 403 が出る**
  - もう一度 **ログアウト → ログイン** し直す（トークンが古い可能性があります）。
  - バックエンドを止めてから `npm run dev` で再起動したか確認する。
- **ログインできない**
  - 管理者アカウントが未作成の場合は、`docs/CREATE_TEST_USER.md` の手順で管理者を作成する。
- **バックエンドが起動しない**
  - `apps/qr-attendance/backend` に `.env` があり、DB の接続情報が正しいか確認する。

---

この手順で、管理者画面の 403 と「User email is required」が解消し、一覧などが表示される想定です。

# 6.4.11・6.4.12 レポート機能 詳細仕様案

**参照**: [デバイス・マトリクス完全版とハイブリッドUI設計指針](FE_DEVICE_MATRIX_AND_HYBRID_UI.md)  
**前提**: 6.4.8〜6.4.10 完了。WBS は既存の 31 行マスターを維持する。

## 1. No. 6.4.11（管理者：出席レポート表示）

### 1.1 概要

イベントごとに **申込数・出席数・欠席数・出席率** を表示する。  
デバイス・マトリクスに基づき、PC では一覧表、スマホ（緊急時）ではカード形式と警告バナーを適用する。

### 1.2 表示項目（必須）

| 項目     | 説明・算出方法 |
|----------|----------------|
| **申込数** | 当該イベントの `registrations` 件数 |
| **出席数** | 当該イベントで打刻（`attendance_logs` に 1 件以上）した人数（重複は 1 人 1 回とみなす） |
| **欠席数** | 申込数 − 出席数（申込済みで未打刻の人数） |
| **出席率** | 申込数 > 0 のとき (出席数 / 申込数) × 100（小数 1 桁）、申込数 0 のとき 0.0% |

※ 既存 API `GET /v1/admin/events/{eventId}/attendance-report` の `summary` で返却済み（`total_registrations`, `total_attendees`, `no_show_count`, `attendance_rate`）。FE はこれをそのまま表示する。

### 1.3 対象画面

- **単一イベント出席レポート**: `/admin/events/[eventId]/attendance-report`  
  - 上記 4 指標のほか、滞在時間統計・時間帯別分布・出席履歴一覧を表示する既存画面を、下記ハイブリッド方針に合わせて改修する。
- **レポート入口（任意拡張）**: `/admin/reports`  
  - イベント一覧に、各イベントの 申込数・出席数・欠席数・出席率 を一覧表示する場合は、同じく PC＝一覧表・スマホ＝カード＋警告バナーとする。

### 1.4 デバイス別表示（ハイブリッドUI）

| 画面幅 | 表示形式 | 備考 |
|--------|----------|------|
| **≥ 768px（PC・タブレット）** | **一覧表（テーブル）** | サマリー 4 指標は表の 1 行または見出し付きテーブルで表示。出席履歴は既存の `<table>` を維持。 |
| **< 768px（スマホ・緊急時）** | **カード形式** | サマリーは 4 指標を 1 イベント 1 カード内に縦積み。出席履歴はテーブルではなくカード（1 人 1 カード）で表示。 |

### 1.5 緊急用モバイル表示の警告バナー

- 画面幅 **< 768px** のとき、当ページでもヘッダー直下に次のバナーを表示する。
- **文言**: 「※現在は緊急用モバイル表示です」
- **スタイル**: オレンジまたは黄系背景、アイコン任意。既存の `EmergencyMobileBanner` 等と統一。
- **≥ 768px** では非表示。

### 1.6 サマリー表示の具体案

- **PC（≥ 768px）**  
  - 4 指標を 1 行で見せるテーブル（例: 列「申込数」「出席数」「欠席数」「出席率」、1 行で数値）か、現状の 4 カラムグリッドを「表形式」に見せるいずれかで「一覧表」とする。
- **スマホ（< 768px）**  
  - 4 指標を 1 カード内に縦に並べる（例: 申込数 / 出席数 / 欠席数 / 出席率 をそれぞれラベル＋数値で表示）。その上に警告バナーを表示。

### 1.7 出席履歴の表示

- **PC**: 既存の `<table>`（ユーザー名・メール・入室時刻・退室時刻・滞在時間・担当者）を維持。
- **スマホ**: `<table>` を非表示にし、同じデータを **カード形式**（1 件 1 カード、縦積み）で表示。タップ領域は 44px 以上を目安とする。

### 1.8 API・権限

- **API**: `GET /v1/admin/events/{eventId}/attendance-report`（既存）
- **権限**: 管理者（`role_flag=3`）のみ。`RoleGuard` で保護済みとする。

## 2. No. 6.4.12（管理者：レポートCSV出力）

### 2.1 概要

「表示されているレポート」を **日本語・BOM 付き UTF-8** の CSV で出力する。

### 2.2 出力仕様

| 項目 | 内容 |
|------|------|
| **文字コード** | UTF-8（BOM 付き）。Excel 等で日本語が文字化けしないようにする。 |
| **言語** | ヘッダー・列名は日本語。 |
| **内容** | 当該イベントの「出席レポート」に対応する一覧（申込者・打刻有無・打刻日時など）。 |

### 2.3 既存 API の利用

- **エンドポイント**: `GET /v1/admin/reports/events/{eventId}/csv`
- **権限**: 管理者のみ（`checkAdminPermission`）。スタッフは 403。
- **レスポンス**: 既に BOM（`\uFEFF`）＋ UTF-8、日本語ヘッダーで返却済み。
  - 例: `イベントID,開催日,イベント名,利用者名,ふりがな,メールアドレス,区分（生徒/一般）,申込日時,打刻日時（実績）`
- **ファイル名**: 例 `event_attendees_{eventId}_{YYYY-MM-DD}.csv`

### 2.4 フロントエンドの責務

- 出席レポート画面（`/admin/events/[eventId]/attendance-report`）に **「CSV をダウンロード」** ボタンを配置する。
- クリック時に `GET /v1/admin/reports/events/{eventId}/csv` を呼び出し、認証ヘッダー（Authorization）を付与する。
- レスポンスの `Content-Disposition` に従い、返却されたバイナリ（BOM＋UTF-8 の CSV）をそのままファイルとして保存・ダウンロードさせる。クライアント側で再エンコードしない。

### 2.5 CSV 内容の整合性

- 「表示されているレポート」＝ 当該イベントの出席レポート画面で見ているイベントの、申込者一覧および打刻実績。
- 既存 API は「申込者ベースで 1 行 1 人、打刻日時あり/なし」を出力しているため、この仕様で「表示されているレポートを CSV 出力」を満たす。必要に応じて退室時刻・滞在時間などを CSV 列に追加する場合は、API と本仕様の両方で列定義を更新する。

### 2.6 エラー・権限

- 403: 管理者以外はボタン非表示または API 呼び出しで 403 を表示。
- 404: イベントなしの場合は既存と同様にエラーメッセージを表示。
- ダウンロード失敗時はトースト等で「CSV のダウンロードに失敗しました」と通知する。

## 3. デバイス・マトリクスとの対応

| 役割 | メイン端末 | サブ（緊急時） | 6.4.11・6.4.12 での適用 |
|------|------------|----------------|---------------------------|
| **管理者** | PC | スマホ | 通常は PC で一覧表・CSV 出力。緊急時はスマホでカード＋警告バナー表示・CSV 出力も可能とする。 |

- 6.4.11: PC＝一覧表、スマホ＝カード＋警告バナー（上記 1.4〜1.7 で規定）。
- 6.4.12: 端末を問わず、表示中イベントの CSV を BOM 付き UTF-8 でダウンロード可能とする。

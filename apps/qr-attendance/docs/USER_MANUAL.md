# QRコード打刻システム 利用マニュアル

**対象**: WBS No.10 マニュアル作成  
**前提**: 結合テスト（WBS No.8）完了済み

---

## 第1部：利用者向けガイド

### 1.1 ログインのしかた

1. ブラウザでシステムのログイン画面を開きます（例: `http://localhost:3000/login`）。
2. **メールアドレス**と**パスワード**を入力します（管理者から事前に案内されたアカウントを使用します）。
3. **「ログイン」**ボタンを押します。
4. ログインに成功すると、**ホーム画面**（/home）が表示されます。

**うまくログインできない場合**
- メールアドレス・パスワードに誤りがないか確認してください。
- 「認証が必要です」「Invalid email or password」と表示される場合は、入力内容を確認するか、管理者にパスワード再設定を依頼してください。

---

### 1.2 お知らせの確認

1. ログイン後、**ホーム画面**の上部に **「重要なお知らせ」** ブロックが表示されます。
2. お知らせが1件以上ある場合、タイトルと本文が表示されます。
3. **重要（緊急）** のお知らせは、赤いバッジや枠で強調表示されます。
4. お知らせが0件の場合は、このブロックは表示されません。

お知らせは「公開中」かつ「掲載期間内」のものだけが表示されます。古いお知らせや非公開のものは表示されません。

---

### 1.3 イベントへの申し込み

1. ホーム画面の **「イベント一覧」** をタップ（またはクリック）します。
2. 一覧から参加したいイベントのカードを選び、**「詳細・申し込む」** をタップします。
3. イベント詳細画面で **「申し込む」** ボタンを押します。
4. 申込が完了すると、成功メッセージが表示されます。
5. **「参加申込一覧」**（メニューまたはホームから）を開くと、申し込んだイベントが一覧で確認できます。

**注意**
- 同じイベントに二重で申し込むことはできません（既に申込済みの場合はエラーになります）。
- 申込後は、当日の打刻用QRを表示してスタッフにスキャンしてもらうか、スタッフによる手動打刻で出席が記録されます。

---

### 1.4 打刻用QRコードの表示（当日の出席用）

1. ホーム画面の **「打刻用QR表示」** をタップします。
2. 画面に **QRコード** が表示されます。
3. イベント当日、受付などで **スタッフにこのQRコードをスキャンしてもらいます**。
4. スキャンが成功すると、入室打刻が記録されます（スタッフ側で確認できます）。

**ポイント**
- QRコードは一定時間で更新されます。読み取れない場合は、画面を再度表示し直して新しいQRを提示してください。
- QRが読めない場合は、スタッフに「手動打刻」で氏名・イベントを指定して打刻してもらうことができます。

---

## 第2部：スタッフ・管理者向け運用マニュアル

### 2.1 QRスキャンによる打刻（スタッフ）

**手順**
1. スタッフまたは管理者アカウントでログインし、**「打刻スキャン（スタッフ用）」** を開きます。
2. 画面上で **イベントを選択** します。
3. 利用者のスマホに表示されている **打刻用QRコード** を、カメラまたはスキャン画面で読み取ります。
4. 読み取りが成功すると、**入室打刻** が記録され、成功メッセージが表示されます。
5. 退出時は、同じ利用者のQRを再度スキャンするか、**手動打刻画面** で「退出打刻」を実行します（後述）。

**注意**
- QRコードの有効期限（約10分）内にスキャンしてください。期限切れの場合は、利用者にQRを再表示してもらってから再度スキャンしてください。
- 同一利用者・同一イベントで「入室」を連打しても、二重には記録されません（1件のみ登録されます）。

---

### 2.2 手動入退室（入室・退出ボタンの使い分け）

QRが読めない場合や、紙の名簿で対応する場合は **「手動打刻」** を使います。

**手順**
1. **「打刻スキャン」** 画面の **「QRが読めない場合はこちら」** から手動打刻画面へ進むか、メニューから **「手動打刻」** を開きます。
2. **イベントを選択** し、**生徒を検索**（2文字以上入力して検索）して、該当する生徒を選択します。
3. 画面に **「現在の状態」** が表示されます。
   - **未打刻**: 入室・退出どちらも未記録。
   - **入室済み（退出未記録）**: 入室のみ記録済み。
   - **入室済み・退出済み**: 入室と退出の両方記録済み。
4. **「入室打刻」** と **「退出打刻」** のどちらかを選んで押します。
   - **入室打刻**: そのイベントへの「入室」を記録します。既に入室済みの場合はエラー（既に打刻済み）になります。
   - **退出打刻**: そのイベントからの「退出」を記録します。既に退出済みの場合はエラーになります。
5. 成功すると、「〇〇 さんの入室打刻を記録しました。」または「〇〇 さんの退出打刻を記録しました。」と表示されます。

**使い分けの目安**
- 受付で来たとき → **入室打刻**
- 帰るとき・退場時 → **退出打刻**
- 同じ種別（入室だけ・退出だけ）を連打しても、2件目は記録されません（二重打刻防止）。

---

### 2.3 出席確認・イベント一覧への戻り方

- **出席確認（参加者一覧）**: イベントごとに「出席確認画面へ」または「参加者一覧」から、そのイベントの参加者と入室・退出時刻を確認できます。
- **「イベント一覧に戻る」**: スタッフの場合は **利用者向けイベント一覧**（/events）へ、管理者の場合は **管理者用イベント一覧**（/admin/events）へ戻ります。どちらも **ログアウトせず** 遷移します。
- **「ホームに戻る」**: **/home** に遷移します。ログイン画面に飛ばされる場合は、リンク先が誤っている可能性があるため、運用担当者に連絡してください。

---

### 2.4 APIエラー時の対応（503 など）

画面にエラーメッセージが表示された場合の対処です。

| 表示・状況 | 想定原因 | 対応 |
|------------|----------|------|
| **「Database is busy (too many connections)」** または **503 エラー** | データベースの接続数が上限に達している。 | しばらく時間をおいてから **再試行** してください。不要なタブや長時間放置した画面を閉じ、同時アクセスを減らすと改善することがあります。 |
| **「Database connection failed」** または **サービス利用不可** | DB が停止している、またはネットワーク・設定の不具合。 | 管理者に連絡し、**バックエンド・DB の起動状況** を確認してもらってください。 |
| **「認証が必要です」「Invalid email or password」**（ログイン時） | メールまたはパスワードの誤り、またはセッション切れ。 | 入力内容を確認し、再度ログインしてください。パスワードを忘れた場合は管理者に **パスワード再設定** を依頼してください。 |
| **「リクエストが無効です」**（400） | 送信したデータの形式や内容が不正。 | 操作内容を確認し、画面の指示に沿って再入力・再操作してください。 |
| **「アクセスが拒否されました」**（403） | 権限のない機能にアクセスしている。 | スタッフ用・管理者用の機能は、それぞれの権限でログインしているか確認してください。 |
| 画面が真っ白、または古い一覧のまま更新されない | 通信エラー後に画面がエラー状態のまま。 | ブラウザで **再読み込み**（F5 またはスワイプ更新）してください。エラーが続く場合は上記の 503・DB 接続の確認を依頼してください。 |

**運用上の注意**
- 503 や DB 接続エラーが続く場合は、**バックエンド（API）やデータベースの再起動** を管理者に依頼してください。
- エラー内容が分かる場合は、**メッセージの文言** や **表示された画面** を控えておくと、管理者の切り分けがしやすくなります。

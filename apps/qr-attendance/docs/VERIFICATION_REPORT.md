## 2. 生成されたCSVをテキストエディタで開いた時の、先頭行のデータ形式

申込数0のイベントで「CSVをダウンロード」したファイルを、**テキストエディタ**（UTF-8で開く設定）で開いたときの構造です。バイナリ上は先頭に BOM が付き、その直後が 1 行目です。

### 2.1 先頭のバイト列（バイナリ）

- **先頭 3 バイト（hex）**: `EF BB BF`  
  - これは UTF-8 の BOM（U+FEFF の UTF-8 表現）。Excel が UTF-8 として認識するために必要です。
- 4 バイト目以降: 1 行目のテキスト（UTF-8）。

### 2.2 1 行目（BOM の直後＝CSV の論理的な 1 行目）

テキストエディタで「1 行目」として見える内容は次のとおりです（BOM は多くのエディタでは見えません）。

```
イベントID,開催日,イベント名,利用者名,ふりがな,メールアドレス,区分（生徒/一般）,申込日時,打刻日時（実績）
```

### 2.3 申込数0のときの CSV 全体の行構成

申込者が 0 人の場合、データ行はなく、末尾にサマリー行が付きます。

| 行 | 内容 |
|----|------|
| 1 | イベントID,開催日,イベント名,利用者名,ふりがな,メールアドレス,区分（生徒/一般）,申込日時,打刻日時（実績） |
| 2 | （空行） |
| 3 | 申込数,出席数,欠席数,出席率（%） |
| 4 | 0,0,0,0.0 |

※ 2 行目は空行、4 行目の出席率は **0.0**（0除算ガードによる）。

### 2.4 テキストエディタでの見え方（再現）

```
イベントID,開催日,イベント名,利用者名,ふりがな,メールアドレス,区分（生徒/一般）,申込日時,打刻日時（実績）

申込数,出席数,欠席数,出席率（%）
0,0,0,0.0
```

※ 先頭に BOM（EF BB BF）が付いているため、エディタで「BOM 付き UTF-8」として開くと日本語が正しく表示されます。

## 3. ブラウザの幅を変えた時の、警告バナーが出る瞬間のクラス適用状態

**EmergencyMobileBanner** は、Tailwind の **`md` ブレークポイント（min-width: 768px）** で表示・非表示が切り替わります。以下は、そのときの「クラス」と「実効スタイル」の対応です。

### 3.1 バナーに指定されているクラス（コード上の定義）

**ファイル**: `frontend/src/components/ui/EmergencyMobileBanner.tsx`

```tsx
className="md:hidden flex items-center gap-2 px-4 py-3 bg-amber-100 border-b border-amber-200 text-amber-900 text-sm"
```

- **md:hidden**: 「min-width: 768px 以上のとき `display: none` を適用」（Tailwind デフォルト）
- それ以外: 指定なし → 要素は表示（ブロックとして表示され、flex は子の並び用）

### 3.2 ビューポート幅ごとの状態

| ビューポート幅 | メディアクエリ (min-width: 768px) | md:hidden の適用 | バナー表示 | 実効スタイル（表示側） |
|----------------|-----------------------------------|------------------|------------|-------------------------|
| **767px**      | 偽（マッチしない）                | 適用されない     | **表示**   | `display: flex` 相当（表示） |
| **768px**      | 真（マッチする）                  | 適用される       | **非表示** | `display: none`         |

### 3.3 「バナーが出る瞬間」のクラス適用状態（文章で）

- **幅 767px のとき**  
  - メディア `(min-width: 768px)` は **偽**。  
  - したがって **`md:hidden` は適用されない**。  
  - バナー用の `div` は表示され、クラスは `md:hidden flex items-center gap-2 ...` のままですが、**この幅では `md:hidden` が効かない**ため、結果としてバナーが表示されます。

- **幅 768px のとき**  
  - メディア `(min-width: 768px)` は **真**。  
  - **`md:hidden` が適用**され、`display: none` になります。  
  - バナーは非表示になります。

### 3.4 開発者ツールで確認する場合

1. 出席レポート画面を開く。
2. 開発者ツールで「※現在は緊急用モバイル表示です」の `div` を選択。
3. ビューポートを **767px** にすると、Computed で `display: flex`（または block）となり、バナーが表示される。
4. ビューポートを **768px** にすると、Computed で `display: none` となり、バナーが消える。

※ ブレークポイントは Tailwind のデフォルト（`md` = 768px）を使用しているため、`tailwind.config.js` で変更していなければ上記の幅で切り替わります。

## 6. 767px / 768px での表示切り替え（クラス適用）の証明

### 6.1 適用方針

- **md ブレークポイント** = 768px（Tailwind デフォルト）。
- **≥768px**: テーブル表示。**<768px**: カード表示＋画面上部に「※現在は緊急用モバイル表示です」バナー。

### 6.2 対象画面と Tailwind クラス

| 画面 | テーブル用ラッパー | カード用ラッパー | バナー |
|------|---------------------|------------------|--------|
| 生徒名簿（6.4.13） | `hidden md:block`（テーブル） | `md:hidden`（カード） | `EmergencyMobileBanner`（md 以上で非表示はコンポーネント内の `md:hidden`） |
| スタッフ管理（6.4.14） | `hidden md:block`（テーブル） | `md:hidden`（カード） | 同上 |
| 参加申込一覧（6.4.15） | `hidden md:block`（テーブル） | `md:hidden`（カード） | 同上 |

### 6.3 クラス適用の意味（767px / 768px）

- **幅 767px のとき**
  - `(min-width: 768px)` は **偽**。
  - `hidden md:block` → `hidden` のみ適用 → テーブルは **非表示**。
  - `md:hidden` → 適用されない → カードは **表示**。
  - バナー用要素（`md:hidden`）→ 適用されない → バナー **表示**。

- **幅 768px のとき**
  - `(min-width: 768px)` は **真**。
  - `hidden md:block` → `md:block` が適用 → テーブルは **表示**。
  - `md:hidden` → 適用 → カードは **非表示**。
  - バナー用要素 → `md:hidden` が適用 → バナー **非表示**。

### 6.4 開発者ツールでの確認手順

1. 生徒名簿・スタッフ管理・参加申込一覧のいずれかを開く。
2. テーブルを包む要素（`hidden md:block`）を選択し、Computed で **767px**: `display: none`、**768px**: `display: block` を確認。
3. カードを包む要素（`md:hidden`）を選択し、**767px**: `display` が none 以外、**768px**: `display: none` を確認。
4. バナー要素で **767px**: 表示、**768px**: `display: none` を確認。

※ ブレークポイントは Tailwind の `md`（768px）を使用。`tailwind.config.js` で変更していなければ上記の幅で切り替わる。

## 8. 未来予約の証明（掲載開始日が未来の場合は1秒でも前なら表示しない）

### 8.1 バックエンドのクエリ条件

`GET /v1/news`（利用者向けお知らせ一覧）では、次の WHERE 条件を**厳密に**適用している。

- `COALESCE(is_published, 1) = 1` … 公開のみ
- **`published_at <= ?`** … サーバー現在時刻（`now`）**以前**の開始日のお知らせのみ。掲載開始日が未来（1分後など）の場合は `published_at <= now` が偽となり、**一切返却されない**。
- `(expired_at IS NULL OR expired_at >= ?)` … 掲載終了前

該当コード（`backend/functions/news/list/index.js`）抜粋:

```javascript
const now = new Date().toISOString().slice(0, 19).replace('T', ' ');
const whereClause = ' WHERE COALESCE(is_published, 1) = 1 AND published_at <= ? AND (expired_at IS NULL OR expired_at >= ?)';
// ...
const [rows] = await db.execute(listSql, [now, now]);
```

### 8.2 検証手順とログ（テキスト再現）

1. **お知らせを1件作成**し、掲載開始日を**現時点の1分後**に設定する（例: 2026-03-19 15:00 に実行する場合、開始日を 2026-03-19 15:01 に設定）。
2. 直後に **GET /v1/news** を呼ぶ（Authorization に有効なトークンを含める）。
3. **レスポンスの `news` 配列に、そのお知らせの `id` が含まれていない**ことを確認する。

**期待される結果のログ（テキスト再現）**:

```
GET /v1/news を実行した時刻: 2026-03-19T15:00:00 前後（例）
作成したお知らせ: id=123, title="テスト未来予約", published_at="2026-03-19 15:01:00"

レスポンス例:
{
  "news": [],
  "totalCount": 0,
  "hasNextPage": false
}
```

または、既に別のお知らせが掲載期間内で存在する場合でも、`id=123` は `news` 配列に**含まれない**。  
→ **開始日が1分後のお知らせは、現時点の API レスポンスに一切含まれない**ことをもって、未来予約の厳密性を証明する。

## 9. 重要バッジの証明（767px・スマホ表示）

### 9.1 利用者ホームでの重要お知らせの見た目

- **種別が「重要（緊急）」**（`announcement_type === 2`）のお知らせは、利用者ホームの「重要なお知らせ」ブロック内で次のように表示される。
- **全幅（767px 含むスマホ）で共通**のクラス適用により、重要お知らせは**赤色のバッジと赤い左枠線**で目立たせている。

### 9.2 テキスト・クラス適用の再現（767px 時）

**表示テキストの例**（重要お知らせ1件の場合）:

```
重要なお知らせ
────────────────────────────────────
│ [重要]  ○○に関するお知らせ
│         本文テキストがここに表示されます。
────────────────────────────────────
```

**適用している Tailwind クラス（重要お知らせの `<li>` およびバッジ）**:

| 要素 | クラス | 役割 |
|------|--------|------|
| 重要お知らせの `<li>` | `border-l-4 border-l-red-500 pl-3` | 左に赤い太枠線（4px）、左パディングで本文を右に寄せる。**767px でも 768px でも同じく適用**。 |
| 「重要」バッジ `<span>` | `inline-flex items-center px-2 py-0.5 rounded text-xs font-medium bg-red-100 text-red-800 border border-red-300` | 赤系背景・赤文字・赤枠の小さなバッジ。「重要」と表示。 |

上記のため、**767px（スマホ）**で重要お知らせは次のように見える。

- 左側に**赤い縦線**（`border-l-4 border-l-red-500`）が表示される。
- タイトル直前に**「重要」**と書かれた**赤色のバッジ**（`bg-red-100 text-red-800 border-red-300`）が表示される。
- 本文はその下にプレーンテキストで表示される。

開発者ツールでビューポートを 767px にし、該当 `<li>` の Computed で `border-left: 4px solid rgb(239 68 68)`（red-500）、バッジの `background-color` が赤系であることを確認すれば、クラス適用の証明となる。

# No.6.4.9・6.4.10 管理者：イベント管理一覧・イベント編集 詳細仕様案

**WBS**: No.6.4.9 管理者：イベント管理一覧、No.6.4.10 管理者：イベント編集  
**目的**: 管理機能の要となるイベント一覧（ハブ画面）と編集フォームの仕様を確定し、ルーティング一貫性・入力バリデーションを明文化する。

---

## 1. ルーティング・一貫性の整理

### 1.1 正とするルート（他画面との矛盾なし）

| 用途 | パス | 備考 |
|------|------|------|
| 管理者イベント管理一覧（ハブ） | `/admin/events` | ホームメニュー「イベント管理」の行き先。新規作成・一覧・各行から編集・削除・QR・出席確認・レポートへ。 |
| 出席確認（参加者一覧） | `/admin/events/[eventId]/participants` | 6.4.8 で整備済み。一覧の「参加者」から遷移。 |
| 出席レポート | `/admin/events/[eventId]/attendance-report` | 一覧の「レポート」から遷移。 |
| ホーム | `/home` | 全画面の「ホームに戻る」はここに統一（既方針）。 |

### 1.2 イベント管理一覧でのリンク先

- **ホームに戻る**: `href="/home"` に統一（現状は `"/"` のため修正対象）。
- **イベント名**: クリック先は次のいずれかで統一する。
  - **案A**: 編集画面（または編集モーダル）に飛ばす（管理ハブとして編集を主とする）。
  - **案B**: 公開側のイベント詳細 `/events/[eventId]` のまま（利用者目線のプレビュー用）。
- **参加者**: `href={/admin/events/${eventId}/participants}` のまま（変更なし）。
- **レポート**: `href={/admin/events/${eventId}/attendance-report}` のまま（変更なし）。

※ 本仕様では **案B（イベント名は公開詳細へ）** を採用し、編集は「編集」ボタンのみとする。必要なら後から案Aに変更可能。

## 2. No.6.4.9 管理者：イベント管理一覧

### 2.1 画面の位置づけ

| 項目 | 内容 |
|------|------|
| **パス** | `/admin/events`（既存。`app/admin/events/page.tsx`） |
| **権限** | 管理者のみ（RoleGuard または layout で制御）。 |
| **役割** | イベントの新規作成・一覧表示のハブ。各行から編集・削除・QR・出席確認・レポートへ遷移。 |

### 2.2 UI構成（ワイヤーフレーム的）

```
┌─────────────────────────────────────────────────────────────────┐
│  イベント管理              [ ホームに戻る ]  [ 新規イベント作成 ]  │
│  イベントの作成・編集・削除ができます                             │
├─────────────────────────────────────────────────────────────────┤
│  （エラー・成功メッセージ）                                       │
├─────────────────────────────────────────────────────────────────┤
│  イベント名 │ 開催日時 │ 場所 │ 定員 │ 操作                      │
│  ──────────┼─────────┼─────┼─────┼──────────────────────────  │
│  ○○勉強会  │ 2026/3/15│ 会議室A│ 30人 │ [編集][QR][参加者][レポート][削除] │
│  △△説明会  │ 2026/3/20│ -     │ -   │ [編集][QR][参加者][レポート][削除] │
├─────────────────────────────────────────────────────────────────┤
│  （0件時）「イベントがありません」＋「最初のイベントを作成」       │
└─────────────────────────────────────────────────────────────────┘
```

- **ヘッダー**: タイトル「イベント管理」、説明文、「ホームに戻る」は **`/home`**、「新規イベント作成」ボタン。
- **一覧**: 表形式で イベント名・開催日時・場所・定員・操作列。操作列は「編集」「QRコード」「参加者」「レポート」「削除」。
- **リンク**: 参加者 → `/admin/events/[eventId]/participants`、レポート → `/admin/events/[eventId]/attendance-report`。ホームに戻る → `/home`。

### 2.3 使用API

| メソッド | エンドポイント | 用途 |
|----------|----------------|------|
| **GET** | `/v1/admin/events?limit=&offset=` | 一覧取得（既存 `listEvents`）。 |
| **POST** | `/v1/admin/events` | 新規作成（既存 `createEvent`）。モーダルから実行。 |
| **PUT** | `/v1/admin/events/[eventId]` | 更新（既存 `updateEvent`）。編集モーダルから実行。 |
| **DELETE** | `/v1/admin/events/[eventId]` | 削除（既存 `deleteEvent`）。 |
| **GET** | `/v1/admin/events/[eventId]/qr` | QR表示用（既存。モーダル内）。 |

## 3. No.6.4.10 管理者：イベント編集

### 3.1 画面の位置づけ

| 項目 | 内容 |
|------|------|
| **実装形態** | 既存どおり **編集モーダル**（一覧の「編集」クリックで開く）。専用ルートは設けない。 |
| **役割** | イベント名・開催日時・場所・定員・概要の入力・更新。管理者ミスを防ぐバリデーションを追加する。 |

### 3.2 フォーム項目

| 項目 | 物理名 | 必須 | 入力形式 | 備考 |
|------|--------|------|----------|------|
| イベント名 | `event_name` | ○ | テキスト | 空不可。 |
| 開催日時 | `event_date` | ○ | datetime-local または日時ピッカー | 過去日時は不可（バリデーション）。 |
| 場所 | `location` | - | テキスト | 任意。 |
| 定員 | `capacity` | - | 数値（正の整数） | 空の場合は「制限なし」。1以上の整数のみ許可。 |
| 概要 | `summary` | - | テキストエリア | 任意。 |

### 3.3 入力バリデーション（管理者ミス防止）

| チェック | ルール | エラー表示例 |
|----------|--------|--------------|
| イベント名 | 必須。空白のみ不可。 | 「イベント名を入力してください」 |
| 開催日時 | 必須。**開催日時は未来の日時にしてください**（新規作成時）。編集時は「過去の日時は許可しない」または「編集時は過去も許可」のどちらかで統一（本仕様では **新規は未来のみ・編集は過去も許可** とする）。 | 「開催日時は未来の日時を指定してください」 |
| 定員 | 未入力の場合は null（制限なし）。入力時は **正の整数** のみ。小数・負数・非数はエラー。 | 「定員は1以上の整数で入力してください」 |

- **新規作成**: 開催日時が現在より前ならエラー。
- **編集**: 既存イベントの日時を「過去に変更」することは業務上あり得るため、編集時は過去日時を許可する（またはプロジェクト方針で「編集も未来のみ」に変更可）。
- クライアント側で上記をチェックし、通過した場合のみ API 呼び出し。API 側でも同様のバリデーションがあれば二重でガード。

### 3.4 使用API

- **PUT** `/v1/admin/events/[eventId]`（既存 `updateEvent`）  
  - ボディ: `event_name`, `event_date`, `location`, `capacity`, `summary`（いずれも仕様に沿った形式）。

## 4. 実装時の整理ポイント

1. **一貫性**
   - `/admin/events` の「ホームに戻る」を **`/home`** に変更。
   - 出席確認（6.4.8）・レポート・ホームのルートは上記のとおり変更しない。

2. **6.4.9**
   - 新規作成ボタン・一覧・編集・削除・QR・参加者・レポートのリンクを維持。
   - WBS コメントをファイル冒頭に追加（No.6.4.9）。

3. **6.4.10**
   - 新規作成モーダル・編集モーダル双方に、上記バリデーションを追加。
   - 定員: 空なら `undefined`/送信しない、数値なら 1 以上整数のみ許可。
   - 開催日時: 新規時は現在より未来であることをチェック。
   - WBS コメントを該当モーダルコンポーネントまたはページ内に明記（No.6.4.10）。

## 5. 追加実装（承認済み）

- **削除ガード**: 削除ボタン押下時に確認ダイアログで「イベント「○○」を削除しますか？この操作は取り消せません。」と対象イベント名を表示。

- **整合性**: `/admin/events` の「ホームに戻る」を `/home` に変更済み。参加者一覧（`/admin/events/[eventId]/participants`）・出席レポート（`/admin/events/[eventId]/attendance-report`）へのリンクは既存のとおり正しく機能することを確認。

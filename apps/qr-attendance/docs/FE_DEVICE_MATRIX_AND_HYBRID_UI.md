## 1. デバイス・マトリクス完全版（前回を破棄し本表を正とする）

| 役割 | メイン端末 | サブ（緊急時） | 設計優先度 | 運用上の位置づけ |
|------|------------|----------------|------------|------------------|
| **利用者** | スマホ | なし | モバイル専用 | 申込・QR表示。指での操作性を最優先。 |
| **スタッフ** | PC / タブレット | スマホ | デスクトップ優先 | 原則は据え置きPC。**停電やPC故障時のみ**スマホで出席確認を継続。 |
| **管理者** | PC | スマホ | デスクトップ優先 | 原則はデスク作業。**出先での緊急なイベント編集時のみ**スマホを利用。 |

### 1.1 スタッフ・管理者のスマホ利用は「緊急時の予備」である

- **メイン**: PC / タブレット。通常業務は大画面で行う。
- **レジリエンス（回復力）**: 現場のトラブル（停電・PC故障・出先）時に、スマホでも**最低限の業務が継続できる**ことを重視する。
- スマホ表示は「緊急用」であることをユーザーに明示し、通常はPC/タブ利用を促す。

## 2. ハイブリッドUIの実装方針

### 2.1 対象画面

**6.4.8 以降のスタッフ・管理者画面**（出席確認、イベント管理一覧、イベント編集、および今後の 6.4.11〜6.4.16 の管理系画面）。

### 2.2 表示形式の切り替え

- **広い画面幅**（目安: `min-width: 768px` 以上、md ブレークポイント）  
  → **テーブル形式**で一覧表示。情報量が多く、PC/タブでの操作に最適。
- **狭い画面幅**（`max-width: 767px`、スマホ幅）  
  → **カード形式**で一覧表示。縦積みで横スクロールを避け、タップしやすくする。

### 2.3 緊急用モバイル表示の警告

- スタッフ・管理者画面で、**スマホ幅のときのみ**、画面上部（ヘッダー直下）に次の警告バナーを表示する。
- **文言**: 「※現在は緊急用モバイル表示です」
- **スタイル**: 目立つが邪魔にならない程度（例: オレンジまたは黄系の背景、アイコン付き、閉じるボタンは任意）。
- 利用者向け画面（6.4.1〜6.4.7）では**表示しない**（利用者はモバイル専用のため）。

### 2.4 ブレークポイントの統一

- **md**: 768px（Tailwind: `md:`）。  
  - `>= 768px` → テーブル＋警告なし  
  - `< 768px` → カード＋警告「※現在は緊急用モバイル表示です」

## 3. リスク管理（スマホ利用時の誤操作防止）

### 3.1 方針

- **破壊的操作**（削除・一括取消など）は、モバイル表示時のみ**確認ステップを一段階増やす**。
- 例: 削除ボタン  
  - **PC/タブ**: 従来どおり 1 回の `confirm`（「イベント「○○」を削除しますか？この操作は取り消せません。」）で実行可否を確認。  
  - **モバイル**:  
    1. 1 回目: 「削除する」ボタン or タップで「本当に削除しますか？ イベント名: ○○」と表示。  
    2. 2 回目: 「はい、削除する」で確定、または従来の `confirm` を 2 回連続で表示する等。

### 3.2 対象アクション

- イベント削除（6.4.9）
- その他、データ削除・一括変更など破壊的操作が入る機能では、モバイル時は二重確認を検討する。

## 4. 6.4.8〜6.4.10 修正案（本指針に基づく）

### 4.1 No.6.4.8 スタッフ：出席確認（参加者一覧）

| 項目 | 内容 |
|------|------|
| **ハイブリッド** | `>= 768px`: **テーブル**（列: 打刻状況・氏名・メール・申込日時・電話・役割）。`< 768px`: **カード**（現状の縦積み）を維持。 |
| **警告バナー** | 画面幅 `< 768px` のとき、ヘッダー直下に「※現在は緊急用モバイル表示です」を表示。 |
| **リスク管理** | 当画面に削除等の破壊的操作はないため、二重確認の追加は不要。 |
| **実装** | 一覧部分を `md:block`（テーブル）と `md:hidden`（カード）で出し分け、または `useMediaQuery` 等で幅を検知し表示を切替。 |

### 4.2 No.6.4.9 管理者：イベント管理一覧

| 項目 | 内容 |
|------|------|
| **ハイブリッド** | `>= 768px`: **テーブル**（現状維持。イベント名・開催日時・場所・定員・操作列）。`< 768px`: **カード**（1 イベント 1 カード。カード内に編集・QR・参加者・レポート・削除ボタンを配置）。 |
| **警告バナー** | 画面幅 `< 768px` のとき、ヘッダー直下に「※現在は緊急用モバイル表示です」を表示。 |
| **リスク管理** | **削除**について、モバイル表示時のみ二段階確認とする。  
  - 1 回目: 削除ボタン押下 → 「イベント「○○」を削除しますか？」（confirm またはモーダル）。  
  - 2 回目: 「はい、削除してよい」等の確定用ボタン or 2 回目の confirm で実行。  
  - PC/タブでは従来どおり 1 回の confirm のまま。 |
| **実装** | 一覧をテーブルとカードで出し分け。`handleDelete` 内で `window.innerWidth < 768`（または useMediaQuery）のときだけ 2 段階 confirm に分岐。 |

### 4.3 No.6.4.10 管理者：イベント編集

| 項目 | 内容 |
|------|------|
| **ハイブリッド** | 編集は**モーダル**のため、一覧の表示形式（テーブル/カード）に従う。モーダル自体は幅に応じてレスポンシブに表示（現状で可）。 |
| **警告バナー** | イベント管理一覧でモバイル表示のときにモーダルを開いた場合、モーダル内の上部に「※現在は緊急用モバイル表示です」を表示してもよい（任意）。または一覧側のバナーのみで統一可。 |
| **リスク管理** | 編集モーダルに削除はない。破壊的操作は 6.4.9 の一覧でのみ行うため、6.4.10 では二重確認の追加は不要。 |

## 5. 共通コンポーネント案

- **緊急用モバイル警告バナー**: スタッフ・管理者画面で `isMobileWidth`（例: `window.innerWidth < 768`）のとき表示する共通コンポーネント。  
  - 例: `<EmergencyMobileBanner />` を各ページの main 直下に配置し、md 以上では `hidden`。
- **削除の二段階確認**: 6.4.9 の `handleDelete` 内で、`isMobileWidth` が true のときは 1 回目 confirm → 2 回目 confirm（または「削除してよい」ボタン）で実行するよう分岐。

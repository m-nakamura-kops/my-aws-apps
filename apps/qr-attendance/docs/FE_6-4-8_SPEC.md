# No.6.4.8 スタッフ：出席確認（参加者一覧） 詳細仕様案

**WBS**: No.6.4.8 スタッフ：出席確認（参加者一覧）  
**目的**: 特定イベントを選択した際に「誰が申し込んでいるか」「誰が打刻済みか」を一覧で確認できる画面。実装前に表示項目・API・モバイル配慮を確定する。

---

## 1. 画面の位置づけ・フロー

| 項目 | 内容 |
|------|------|
| **パス** | `/admin/events/[eventId]/participants`（既存） |
| **遷移** | 管理者：イベント管理一覧（`/admin/events`）またはイベント詳細から「参加者」リンクで遷移。スタッフ・管理者は全参加者の打刻状況、利用者は自身の打刻状況のみ閲覧（権限マトリクス既存どおり）。 |
| **役割** | イベント別の**参加者一覧**と**打刻状況（入室・退室時刻）**の確認。スタッフが現場で Android 端末から「誰が来たか」を素早く確認する用途を想定。 |

---

## 2. 一覧の表示項目（案）

### 2.1 必須表示項目

| 表示項目 | 説明 | データソース |
|----------|------|--------------|
| **氏名（漢字）** | 参加者名 | `name_kanji` |
| **氏名（かな）** | ふりがな | `name_kana` |
| **メールアドレス** | 本人特定用 | `email` |
| **申込日時** | いつ申し込んだか | `registration_date` |
| **打刻状況** | 入室・退室の有無と時刻 | 下記「打刻情報」 |

### 2.2 打刻状況の表示

- **打刻あり**: 「✓ 入室 10:30」など入室時刻を表示。退室があれば「退室 12:00」も表示（または「入室 10:30 / 退室 12:00」の1行表記）。
- **打刻なし**: 「未打刻」と表示し、視認性のため色分け（例: 未打刻はグレーまたはオレンジ、打刻済は緑系）。

### 2.3 補助項目（モバイルでは折りたたみ or 2行目）

| 表示項目 | 説明 | 備考 |
|----------|------|------|
| 電話番号 | `tel` | あれば表示。現場連絡用。 |
| 組織ID | `org_id` | あれば表示。 |
| 役割 | 利用者/スタッフ等/管理者 | `role_flag` から表示。 |

※ 現場での「誰が来たか」を最優先にするため、**氏名・申込日時・打刻状況**を前面にし、電話・組織・役割はカード内の補助情報とする。

## 3. 使用API案

### 3.1 現状

- **GET /v1/admin/events/{eventId}/participants**  
  - 戻り: `event_id`, `event_name`, `event_date`, `participants[]`, `pagination`  
  - `participants[]`: `email`, `name_kanji`, `name_kana`, `tel`, `org_id`, `role_flag`, `registration_date`  
  - **打刻情報（in_time, out_time）は含まれていない。**

### 3.2 推奨案：API 拡張（1リクエストで完結）

**GET /v1/admin/events/{eventId}/participants** のレスポンスに、参加者ごとの打刻情報を追加する。

- **拡張後の participants 1件の形（案）**  
  - 既存: `email`, `name_kanji`, `name_kana`, `tel`, `org_id`, `role_flag`, `registration_date`  
  - **追加**: `in_time` (string | null), `out_time` (string | null)  
  - バックエンド: 既存の参加者取得クエリに、`attendance_logs` を **LEFT JOIN**（`event_id` と `email` で結合）し、`in_time`, `out_time` を SELECT。打刻がなければ null。

- **利点**: 1回のAPIで「申込一覧＋打刻状況」が揃う。モバイルの通信回数削減・実装が単純。

### 3.3 代替案：FE で2APIを結合（バックエンド変更なし）

- **GET /v1/admin/events/{eventId}/participants** … 申込者一覧（現状のまま）
- **GET /v1/admin/events/{eventId}/attendance-report** … 打刻一覧（`attendance_logs[]` に `email`, `in_time`, `out_time` 等）

フロントで、`attendance_logs` を `email` をキーにマップ化し、各 `participants` の `email` に応じて入室・退室時刻を紐付けて表示する。

- **利点**: バックエンド変更不要。  
- **欠点**: 2リクエスト、通信遅延時の考慮（先に participants を出し、attendance は後から重ねて表示する等）が必要。

## 4. UI構成（ワイヤーフレーム的）・モバイル配慮

### 4.1 Android 端末での利用を想定した方針

- **カード形式（縦 Stack）**: 表形式は横スクロールが発生しやすく、現場のスマホでは見づらいため、**1参加者＝1カード**の縦積みにする。
- **打刻状況を最前面**: 各カードの上部に「✓ 入室 10:30 / 退室 12:00」または「未打刻」を大きく表示。色で一目で区別（打刻済み＝緑系、未打刻＝グレー or オレンジ）。
- **タップエリア**: ボタン・リンクは **min-height: 44px** 以上、十分な padding を確保。
- **フォント・余白**: 小さな画面でも読みやすいよう、重要なテキストは `text-base` 以上とし、カード内の余白を十分に取る。

### 4.2 ワイヤーフレーム（モバイル優先）

```
┌─────────────────────────────────────────────────────────────────┐
│  ← イベント一覧に戻る               [ 出席レポート ]（管理者のみ） │
```

- **ヘッダー**: タイトル「参加者一覧」、イベント名・開催日時。戻るリンクは「イベント一覧に戻る」（スタッフ/管理者は `/admin/events`、利用者は `/events` または `/user/events`）。管理者のみ「出席レポートを見る」ボタン（`/admin/events/[eventId]/attendance-report`）。
- **カード**: 1参加者1カード。先頭に打刻状況（入室・退室時刻 or 未打刻）、続けて氏名・メール・申込日時・電話・役割。打刻済み/未打刻で背景色またはバッジで区別。
- **ページネーション**: 「もっと見る」は 44px 以上のタップ領域で配置。

## 5. 表示ルールの整理

| 条件 | 表示 |
|------|------|
| 参加者に打刻あり（`in_time` あり） | 「✓ 入室 HH:mm」を表示。`out_time` があれば「退室 HH:mm」も表示。 |
| 参加者に打刻なし（`in_time` なし） | 「未打刻」と表示。 |
| 利用者本人が閲覧 | 自分1件のみ表示（既存の権限マトリクスどおり）。 |

---

## 6. 共通・権限

- **認証**: 未認証は `/login` へ。
- **権限**: スタッフ・管理者は全参加者＋打刻状況を表示。利用者は自身の1件のみ（既存 RoleGuard と API のスコープ分岐を維持）。
- **WBS**: 当該ページのファイル冒頭に「No.6.4.8 スタッフ：出席確認（参加者一覧）」である旨をコメントで明記。

---

## 7. 追加実装（承認済み）

- **API**: 推奨案を採用。バックエンドで `v_event_participants` に `attendance_logs` を LEFT JOIN し、`in_time`, `out_time` を返すように修正済み。
- **現場用検索・フィルタ**: 一覧最上部に「氏名（かな）でのリアルタイム検索窓」と「未打刻者のみ表示」トグルを設置。クライアント側でフィルタ（useMemo）し、約100名でも検索・スクロールがスムーズに動作するよう実装。
- **テスト**: Android実機での視認性確認および「参加者約100名での検索・スクロールの簡易検証」を報告に含める。

## 1. No.6.4.6 参加申込一覧

### 1.1 画面パス・役割

| 項目 | 内容 |
|------|------|
| **パス** | `/user/registrations`（既存。ホームメニュー「参加申込一覧」とマイページ「参加申込一覧を見る」から遷移） |
| **役割** | 自分が申し込んだイベントの一覧表示と、**申込取消**の操作 |

### 1.2 UI構成（ワイヤーフレーム的説明）

**実機環境（Android）前提**: 中村氏による実機検証は Android 端末メイン。画面幅に依存せず、モバイルでも情報欠落・操作性低下を防ぐため **一覧はカード形式（Stack）に統一** する。

```
┌─────────────────────────────────────────────────────────────────┐
│  参加申込一覧                        [ ホームに戻る ]             │
│  あなたが申し込んだイベントの一覧です                             │
├─────────────────────────────────────────────────────────────────┤
│  （エラー表示エリア：取得失敗・取消失敗時のみ）                     │
├─────────────────────────────────────────────────────────────────┤
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 【カード1】                                                │  │
│  │   ○○勉強会                                                 │  │
│  │   開催: 2026/3/15(日) 10:00  場所: 会議室A  定員: 30人     │  │
│  │   申込日時: 2026/3/1 12:00                                │  │
│  │   [ 詳細を見る ]  [ 参加申込を取消 ]  ← タップ領域 44px 以上 │  │
│  └───────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │ 【カード2】  ...                                           │  │
│  └───────────────────────────────────────────────────────────┘  │
│  （件数表示・「もっと見る」は従来どおり。ボタンは min-h 44px）     │
└─────────────────────────────────────────────────────────────────┘
```

- **ヘッダー**: タイトル「参加申込一覧」、説明文、右に「ホームに戻る」ボタン（`href="/home"`）。ボタンは **min-height: 44px** 以上でタップ領域を確保。
- **一覧**: **カード形式（縦 Stack）**。1申込＝1カード。カード内にイベント名・開催日時・場所・定員・申込日時を表示し、最下部に「詳細を見る」「参加申込を取消」を並べる。
- **操作**: 「詳細を見る」は `/events/[eventId]` へのリンク。「取消」はボタン → 確認ダイアログ「参加申込を取消しますか？」→ OK で DELETE API → 成功時は当該カードを一覧から削除（または一覧再取得）。**ボタン・リンクは min-height 44px / padding 十分** でタップしやすくする。
- **0件時**: 「申し込んだイベントはありません」＋「イベント一覧を見る」リンク（`/events`）。ホームに戻るは `/home` に統一。

### 1.3 使用API

| メソッド | エンドポイント | 用途 |
|----------|----------------|------|
| **GET** | `/v1/users/registrations` | 自分の申込一覧取得。クエリ: `email`（必須・トークン由来）, `limit`, `offset`。レスポンス: `registrations[]`, `pagination`。 |
| **DELETE** | `/v1/users/events/{eventId}/register?email={email}` | 申込取消。クエリに `email`（本人）。成功時 200、該当レコードが DB の `registrations` から削除される。 |

- 既存の `apiClient.getRegistrations({ email, limit, offset })` および `apiClient.unregisterFromEvent(eventId, email)` を利用。

### 1.4 出席状況の判定ロジック

- 本画面は「**申込一覧**」のみを扱う。出席（打刻）状況の表示は行わない。
- 申込取消により **DB の `registrations` から該当行が削除される**ことのみ保証する（取消ボタン → DELETE API → 一覧から削除／再取得）。

## 2. No.6.4.7 スケジュール

### 2.1 画面パス・役割

| 項目 | 内容 |
|------|------|
| **パス** | `/schedule`（新規ページ。ホームメニューに「スケジュール」を追加し、申込イベント＋打刻状況を月別で表示） |
| **役割** | 申し込んだイベントを月単位で一覧し、**各イベントの出席（打刻）有無**を表示する。 |

※ `/user/events` は既存で「イベント一覧」リンク先として使われているため、スケジュール専用は `/schedule` とする案。

### 2.2 UI構成（ワイヤーフレーム的説明）

**実機環境（Android）前提**: モバイル表示で情報欠落・操作性低下を防ぐため **一覧はカード形式（Stack）に統一**。ボタン・リンクはタップ領域を十分に確保する。

```
┌─────────────────────────────────────────────────────────────────┐
│  スケジュール                          [ ホームに戻る ]             │
│  申し込んだイベントと打刻状況（月別）                               │
├─────────────────────────────────────────────────────────────────┤
│  [ ◀ 前月 ]  2026年3月  [ 翌月 ▶ ]  ← タップ領域 44px 以上        │
├─────────────────────────────────────────────────────────────────┤
│  （エラー表示エリア：API失敗時のみ）                                │
├─────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 【カード1】                                                  │ │
│  │   2026/3/15(日) 10:00  ○○勉強会                             │ │
│  │   場所: 会議室A  申込: ✓  出席: ✓ 出席済み                   │ │
│  │   [ 詳細を見る ]  ← カード全体 or 明確なリンク（44px 以上）   │ │
│  └─────────────────────────────────────────────────────────────┘ │
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │ 【カード2】  2026/3/20  △△説明会  出席: 未打刻  ...          │ │
│  └─────────────────────────────────────────────────────────────┘ │
│  （0件時）「この月の申込イベントはありません」                     │
└─────────────────────────────────────────────────────────────────┘
```

- **ヘッダー**: タイトル「スケジュール」、説明文、右に「ホームに戻る」（`/home`）。ボタンは **min-height: 44px** 以上。
- **表示月**: クエリ `month=YYYY-MM` で API に渡す。初期値は**当月**。**前月・翌月ボタン**で月を切り替え（セレクトでも可）。ボタンは **min-height: 44px** 以上でタップしやすくする。
- **一覧**: **カード形式（縦 Stack）**。1イベント＝1カード。カード内に開催日時・イベント名・場所・申込（✓）・出席(打刻)（「✓ 出席済み」または「未打刻」）を表示。イベント名または「詳細を見る」で `/events/[eventId]` へ。**リンク・ボタンはタップ領域 44px 以上**。

### 2.3 使用API

| メソッド | エンドポイント | 用途 |
|----------|----------------|------|
| **GET** | `/v1/users/schedule?month=YYYY-MM` | 指定月の「申込イベント＋打刻有無」取得。`month` 省略時は**当月**。認証必須（本人のみ）。 |

**レスポンス例**（既存バックエンド仕様に準拠）:

```json
{
  "schedule": [
    {
      "event_id": 1,
      "event_name": "○○勉強会",
      "event_date": "2026-03-15T10:00:00.000Z",
      "location": "会議室A",
      "capacity": 30,
      "summary": "...",
      "is_registered": true,
      "is_attended": true
    },
    {
      "event_id": 2,
      "event_name": "△△説明会",
      "event_date": "2026-03-20T14:00:00.000Z",
      "location": null,
      "capacity": null,
      "summary": null,
      "is_registered": true,
      "is_attended": false
    }
  ],
  "month": "2026-03"
}
```

- フロントでは **`getSchedule(month?: string)`** を api-client に追加し、上記レスポンスを利用する。

### 2.4 出席状況の判定ロジック

- **判定はバックエンドで実施**。フロントは API の `is_attended` をそのまま表示に使う。
- **バックエンドのロジック**（既存実装）:
  - `registrations` と `events` を JOIN し、自分（`email`）が申し込んだイベントを対象とする。
  - 指定月の `event_date` 範囲でフィルタ（`BETWEEN 月初  AND 月末`）。
  - `attendance_logs` を `event_id` と `email` で LEFT JOIN し、**1件でも打刻があれば `is_attended: true`**、なければ `false`。
- フロントの表示ルール:
  - `is_attended === true` → 「✓ 出席済み」（または「打刻済」）
  - `is_attended === false` → 「未打刻」
